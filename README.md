# 🔋 PEFT for Abstractive Summarization
### Сравнительное исследование параметрически-эффективных методов дообучения ruT5-base

📌 **Описание проекта**

В этом исследовании я сравнил классический full fine-tuning и параметрически-эффективные методы (PEFT) для задачи абстрактивной суммаризации новостей на русском языке, чтобы понять:
- Насколько сокращается число обучаемых параметров и время обучения при использовании адаптеров
- Сохраняется ли качество генерации (по метрикам ROUGE) при сильном сокращении параметров
- Как разные стратегии инжекта адаптеров (в какие модули) влияют на результат

**Датасет**: `gazeta` — новостные статьи и их краткие содержания с портала Газета.Ru, ~5000 примеров для обучения, 500 для валидации. Источник: [IlyaGusev/gazeta](https://huggingface.co/datasets/IlyaGusev/gazeta).

**Базовая модель**: `ai-forever/ruT5-base` — предобученная seq2seq-модель на русском языке, архитектура T5, ~300M параметров.

---

**Методы**:
  - **Baseline**: Full fine-tuning (все параметры обучаемые)
  - **LoRA** (Low-Rank Adaptation): аппроксимация приращений весов матрицами низкого ранга $W + \Delta W = W + \frac{\alpha}{r}BA^T$
    - `r = 4`, `α = 16`, инжект в `[q, v]` (библиотечная реализация)
    - `r = 16`, `α = 16`, инжект в `[q, v]` (библиотечная реализация)
    - Кастомная реализация: `r = 4`, инжект в `[q, v]` и отдельно в `[q, k, v]`
  - **(IA)³** (Infused Adapter by Inhibiting and Amplifying): обучаемые векторы-гейты для рескейлинга активаций
    - Инжект в `[k, v]` и отдельно в `[k, v, wo]`

---

🎯 **Цели исследования**

1. **Реализовать LoRA с нуля**: написать собственный `LoRALayerWrapper`, проверить forward/backward против `PEFT`
2. **Сравнить эффективность обучения**: измерить время тренировки и число обучаемых параметров для каждого метода
3. **Оценить качество суммаризации**: использовать метрики ROUGE-1/2/L/Lsum на валидационной выборке
4. **Сравнить с предобученной моделью**: протестировать на реальных новостях против `IlyaGusev/rut5_base_sum_gazeta`
